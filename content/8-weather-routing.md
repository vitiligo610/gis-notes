## Weather Routing
Weather routing refers to navigating through a complex weather system. The sea vessels (sailboats) speed up and slow down according to the wind speed and direction. Sailing upwind, downwind, or reaching all have quite different performance characteristics for a sailboat. We must maximize the advantage of the wind and use it for improving efficiency.

When using weather routing system, the final path is not a straight, rather it is created while analysing the wind speed and direction and how it affects boat speed as we progress over time.
## Wind Barbs
Wind speeds are a combination of long/short barbs and penants. They graphically depict the speed of wind and its direction. A calm wind is denoted by a simple circle.

One long barb is used to indicate each 10 knots with the short barb representing 5 knots. At 50 knots, the barbs changes to a pennant. For wind speeds higher than 50 knots, long and short barbs are used again in combination with the pennant(s). (See examples below.)

## Polar Plots
The weather routing system needs a model for the vessels performance in various wind conditions. This is called a _polar diagram_. A polar plot (or polar diagram) is a performance graph for a sailboat that illustrates its maximum potential speed at various angles to the wind. Each boat has a unique polar plot which can be generated by Velocity Predictor Program (VPP) based on its weight, hull shape, and sail plan.

A polar diagram can be used to determine the vessels speed, given the true wind angle and speed (TWA, TWS.) For example, a sailboat will have different speeds if it is sailing close hauled, beam reaching or running with the wind.

## Isochrones: Solving the Routing Algorithm
Finding the optimum route through a dynamic weather system is a complex computational problem. If a computer tried to calculate every possible heading, at every time interval, across a multi-day voyage, the number of possible positions would grow exponentially. Within a few hours of simulated travel, the data points would reach into the trillions, making a "brute force" calculation completely impossible. 

To solve this, weather routing software uses **Isochrones**.
### What is an Isochrone?
An isochrone is a line connecting a set of geographic points that can all be reached at the exact same time (a line of "constant time"). 

Instead of tracking every single possible position a boat could end up in, the routing algorithm only looks for the points that **expand the boundary the furthest**. By connecting these outermost points, the software draws an isochrone—representing the maximum possible progress the vessel could make in that specific time step.
![[Pasted image 20260620211930.png]]
### How Isochrones Help in Weather Routing
By relying on isochrones, the algorithm reduces an impossibly large exponential problem into a highly efficient, solvable one. 

The process works like this:
1. **Generate the First Generation:** From the start point, the algorithm uses the wind forecast and the boat's Polar Plot to project where the boat could be in a set time interval (e.g., 20 minutes), simulating multiple headings.
2. **Draw the Boundary:** It discards all the slow, inefficient positions in the middle of the pack and only keeps the points that traveled the furthest. It connects these points to form the first isochrone.
3. **Repeat from the Boundary:** For the next 20-minute jump, it only calculates new positions starting from the points *on that isochrone line*. 
4. **Trace the Path Back:** Every point on an isochrone remembers exactly which point from the previous isochrone it originated from. Once an isochrone finally crosses the destination target, the software simply traces the line backward from the destination, point-by-point, all the way to the start. This creates the optimal weather route.

> [!Note]
> 
>  Because the algorithm tracks the entire "solution space" of isochrones, it can also easily display alternate, near-optimal paths. These alternate paths help the navigator see areas where the routing is less sensitive to minor course deviations.
